# Issuer Competence and Transfer Semantics

## Competence matrix

| Relationship | Competent issuer roles |
|---|---|
| developed_by | developer, authorized publisher |
| published_by | publisher |
| deployed_by | deployer, governance authority |
| operated_by | operator, governance authority |
| controlled_by | current controller, governance authority |
| accountable_to | accountable entity, governance authority |
| assured_by | recognized assurance provider |
| governed_by | governance authority |
| acts_for | principal |
| recognized_by | recognizing governance authority |

Registry administration permission is not universal competence to issue these claims.

## Change classes

Implementations distinguish record-administration transfer, controller change, operator change, economic acquisition, legal succession, custodial transfer and agent supersession. Each affected assertion receives an explicit transition result.

Delegations, assurance claims, runtime credentials, accountable-entity status, sponsor endorsements, confidential evidence access, governance recognition and downstream authority require reissuance, novation or review unless their own terms explicitly preserve validity.
